◆ AWS_IOT_JOBS_FLAG_WAITABLE

#define AWS_IOT_JOBS_FLAG_WAITABLE   ( 0x00000001 )

Allows the use of AwsIotJobs_Wait for blocking until completion.

This flag is only valid if passed to the functions AwsIotJobs_GetPendingAsync, AwsIotJobs_StartNextAsync, AwsIotJobs_DescribeAsync, or AwsIotJobs_UpdateAsync.

An AwsIotJobsOperation_t MUST be provided if this flag is set. Additionally, an AwsIotJobsCallbackInfo_t MUST NOT be provided.

When this flag is set, AwsIotJobsRequestInfo_t.mallocResponse must be set to a function that can be used to allocate memory to hold an incoming response.

Note
If this flag is set, AwsIotJobs_Wait MUST be called to clean up resources.

◆ AWS_IOT_JOBS_FLAG_KEEP_SUBSCRIPTIONS

#define AWS_IOT_JOBS_FLAG_KEEP_SUBSCRIPTIONS   ( 0x00000002 )

Maintain the subscriptions for the Jobs operation topics, even after this function returns.

This flag is only valid if passed to the functions AwsIotJobs_GetPendingAsync, AwsIotJobs_StartNextAsync, AwsIotJobs_DescribeAsync, or AwsIotJobs_UpdateAsync, and their blocking versions.

The Jobs service reports results of Jobs operations by publishing messages to MQTT topics. By default, the Job operation functions subscribe to the necessary topics, wait for the Jobs service to publish the result of the Jobs operation, then unsubscribe from those topics. This workflow is suitable for infrequent Jobs operations, but is inefficient for frequent, periodic Jobs operations (where subscriptions for the Jobs operation topics would be constantly added and removed).

This flag causes the Jobs operation functions to maintain Jobs operation topic subscriptions, even after the function returns. These subscriptions may then be used by a future call to the same function.

This flags only needs to be set once, after which subscriptions are maintained and reused for a specific Thing Name and Jobs function. The function AwsIotJobs_RemovePersistentSubscriptions may be used to remove subscriptions maintained by this flag.

◆ AWS_IOT_JOBS_FLAG_REMOVE_GET_PENDING_SUBSCRIPTIONS

#define AWS_IOT_JOBS_FLAG_REMOVE_GET_PENDING_SUBSCRIPTIONS   ( 0x00000001 )

Remove the persistent subscriptions from a Jobs get pending operation.

This flag is only valid if passed to the function AwsIotJobs_RemovePersistentSubscriptions.

This flag may be passed to AwsIotJobs_RemovePersistentSubscriptions to remove any subscriptions for a specific Thing Name maintained by a previous call to AwsIotJobs_GetPendingAsync or AwsIotJobs_GetPendingSync.

Warning
Do not call AwsIotJobs_RemovePersistentSubscriptions with this flag for Thing Names with any in-progress Jobs get pending operations.
